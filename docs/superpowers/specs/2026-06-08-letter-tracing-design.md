# Kids Playground — Letter Tracing (V1) Design

**Date:** 2026-06-08
**Status:** Approved design, pre-implementation
**Author:** Kshitija + Claude

## Purpose

A free, ad-free, mobile-first web playground for preschoolers (the primary
tester is the author's daughter). Existing kids' apps are paywalled or
ad-ridden; this is a delightful, distraction-free alternative.

V1 ships **one polished activity — Letter Tracing** — good enough to delight a
real toddler and start the build → test-on-child → tune loop. All later
activities (number tracing, patterns, guided drawing) reuse this engine.

## Locked Decisions

| Decision | Choice |
|---|---|
| Stack | Vite + React + TypeScript + Tailwind |
| Hosting | Vercel, pure client-side (no backend, no accounts) |
| Art | Emoji + CSS/SVG shapes, behind a swappable asset layer |
| Home screen | Grid of big tiles (only Letters active in V1) |
| Audio | Deferred — designed around, easy to add later |
| Progress saving | Deferred — in-session state only for V1 |
| Mascot guide | Deferred past V1 |
| Reward model | Per-letter sets, cycled in a fixed predictable order |

## Quality Bar (non-negotiable)

This must feel like a **store-quality app**, not a hobby project — the child is
used to commercial products and it should not feel "icky" by comparison.

- **Polish & feel.** Smooth 60fps animations, no jank, instant load, app-like
  full-screen feel. Installable as a **PWA** (home-screen icon, name, splash,
  offline-capable) so it behaves like a native app, not a web page.
- **Child-safe.** No ads, no external links, no accidental exits, no surprise
  navigation. Nothing that can take her out of the experience.
- **Works on any device.** iOS Safari **and** Android Chrome; phones **and**
  tablets; portrait **and** landscape; small and large screens. Responsive
  canvas, touch-first targets, safe-area / notch handling. Verified on real
  mobile, not just desktop resize.

## Working Style

- **Collaborate on child-level creative calls** (faces, colors, what feels fun)
  — ask before deciding those.
- Keep it tight: don't over-detail; decide technical/implementation specifics
  without ceremony.

## The Letter Tracing Activity

- **Canvas-based tracing.** Faint dotted outline of a letter + a green "start
  here" dot. Finger/mouse follows the stroke.
- **Trace-to-reveal.** The traced path paints in with color as she goes, giving
  visible progress.
- **Forgiving hit-detection.** Generous tolerance for imprecise toddler fingers.
  **No fail state** — she cannot "lose."
- **Reward = the letter's own personality set.** On completion the letter comes
  alive with the next animated personality from **that letter's own set**, cycled
  in a fixed, predictable order (see below).
- **Letter Zoo.** Completed letters collect on a shelf she can revisit and
  re-tap to advance that letter through its set. In-session only for V1 —
  resets on reload (acceptable for V1).
- **Coverage.** A–Z uppercase first; lowercase is a fast follow on the same
  engine.

## Reward Personalities (per-letter sets)

Each letter owns its **own curated set** of animated personalities (each a
color + silly-face + motion combo). On completion or replay, the letter plays
the **next personality in its set, cycled in a fixed order** — so it is
**predictable and anticipatable**: the child learns "the A ones" and looks
forward to them. Variety comes from each letter having a handful of variants;
predictability comes from the fixed cycle order (not random).

Personalities are built from a shared library of reusable animation
"behaviours" (pure CSS), so a letter's set is just an ordered list of
`{ behaviour, color }` entries. Example:

- **A's set:** red googly-dance → orange silly-tongue → pink star-bounce → (repeat)
- **B's set:** blue star-bounce → green googly-dance → purple silly-tongue → (repeat)

Initial behaviour library (extensible without touching the tracing engine):
1. **Googly dance** — wobbly googly eyes, side-to-side jig.
2. **Silly tongue-out** — blinking eyes + lolling tongue, wobble.
3. **Star-eyes bounce** — sparkly star eyes, vertical bounce.

Each letter's set (which behaviours, which colors, what order) lives in
`letterData` alongside its stroke path, so sets are easy to curate. Sound (a
cheerful "boing") is a later add.

## Architecture

Small, isolated units with clear interfaces:

- **`TracingCanvas`** — the reusable engine. Input: a stroke-path definition +
  tolerance. Handles pointer/touch input, trace-to-reveal rendering, and emits a
  `completed` event. Reused later by numbers & guided drawing.
- **`letterData`** — per-letter SVG stroke-path data + start point + the
  letter's **ordered personality set** (`{ behaviour, color }[]`). Generated by
  Claude; stroke order sanity-checked by the author against how the child is
  taught.
- **`RewardPersonality`** — renders one personality given a `{ behaviour, color }`;
  behaviours come from a shared animation library. A small per-letter cursor
  advances through that letter's set in fixed order.
- **`LetterZoo`** — the collected-letters shelf; re-tap replays a reward.
- **`HomeGrid`** — the big-tile menu (Letters active; others as "coming soon"
  tiles).
- **`assets`** — emoji/shape layer, swappable for richer art later.

### Data flow

`HomeGrid` → selects Letters → `LetterTracingScreen` picks next letter →
`TracingCanvas` renders outline from `letterData` → user traces → `completed`
event → `RewardPersonality` plays the next entry in that letter's set → letter
added to `LetterZoo` → next letter.

## Error / Edge Handling

- No fail state; incomplete traces simply don't trigger the reward.
- Pointer events normalized across touch + mouse.
- Canvas sized responsively to the device; large touch targets throughout.
- No network calls → no network error surface in V1.

## Testing

- **Unit:** hit-detection tolerance, completion detection, per-letter personality
  cursor (advances in fixed order, wraps around, independent per letter).
- **Component:** `TracingCanvas` emits `completed` when a stroke is covered
  within tolerance; `LetterZoo` adds/replays.
- **Manual (the real test):** author runs it on the child's device and reports
  what bored vs. delighted her; tune from there.

## Out of Scope for V1

Number tracing, pattern/"what comes next", guided drawing, audio/voice,
progress saving & per-child profiles, mascot guide, lowercase (fast follow),
richer custom art.

## Success Criterion

The child chooses to trace several letters in a sitting and re-taps the Letter
Zoo to replay rewards — i.e. she comes back to it on her own.
